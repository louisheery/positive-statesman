7class Local: 

8 """ 

9 A drop-in replacement for threading.locals that also works with asyncio 

10 Tasks (via the current_task asyncio method), and passes locals through 

11 sync_to_async and async_to_sync. 

12 

13 Specifically: 

14 - Locals work per-coroutine on any thread not spawned using asgiref 

15 - Locals work per-thread on any thread not spawned using asgiref 

16 - Locals are shared with the parent coroutine when using sync_to_async 

17 - Locals are shared with the parent thread when using async_to_sync 

18 (and if that thread was launched using sync_to_async, with its parent 

19 coroutine as well, with this working for indefinite levels of nesting) 

20 

21 Set thread_critical to True to not allow locals to pass from an async Task 

22 to a thread it spawns. This is needed for code that truly needs 

23 thread-safety, as opposed to things used for helpful context (e.g. sqlite 

24 does not like being called from a different thread to the one it is from). 

25 Thread-critical code will still be differentiated per-Task within a thread 

26 as it is expected it does not like concurrent access. 

27 

28 This doesn't use contextvars as it needs to support 3.6. Once it can support 

29 3.7 only, we can then reimplement the storage more nicely. 

30 """

52 # If we're thread-critical, we stop here, as we can't share contexts. 

53 if self._thread_critical:

55 # Now, take those and see if we can resolve them through the launch maps

75 """
